Creating Links

Links in the system are created using a custom syntax. It is generally of the form of:

[target] or [link text|target]

Where "target" is the thing you are linking to, and "link text" is any custom text to be displayed. The "link text" portion is not required.

In the following examples "link text|" is not required. If you leave this off, then the system will use the title of the page/content for the text of your link.

External Links

Links to URLs outside of the web server. You can use full URLs.

[link text|full URL of external webpage] [Monroe County|http://www.co.monroe.in.us]

Relative Links

To link to website content on the same server, but not contained in the content manager application, you can use relative URLs. The URL must start with a slash, "/", signifying the root of the webserver.

[link text|/path to file from website root] [Community Profile|/profile.php]

Email Address

[link text|mailto:email@address.com] [Click to email John Smith|mailto:johnsmith@gmail.com]

Create an Anchor

[anchor:somename] [anchor:section2]

Create a Link to that Anchor

[link text|#section2]

Internal Links

Links to content contained in the content manager. Items in the content manager can be linked to directly:

Documents (links to a page within the site)

You can link to any document in the system by Title, Alias, or ID number. Document links do not need a prefix on the target portion.

[link text|document] or just [document] Examples: [The homepage of the ITS department|Information and Technology Services] [The homepage of the ITS department|ITS] [The homepage of the ITS department|32] [Information and Technology Services] [ITS] [32]

Media

You can link to any media in the system by filename or by ID number. The media does not have to be attached to a document to be able to create a link to the media. Media links use a prefix, "file:" on the target portion.

[link text|file:media] or just [file:media] Examples: [2007-08-01 Plan Commission Minutes|file:pcminutes_20070801.pdf] [2007-08-01 Plan Commission Minutes|file:6] [file:pcminutes_20070801.pdf] [file:6]

Locations

You can create a link to a Location by name or by ID number. Links to locations use "location:" in the target portion.

[link text|location:location] or just [location:location] Examples: [The Animal Shelter|location:Animal Shelter] [The Animal Shelter|location:12] [location:Animal Shelter] [location:12]

Location Groups

You can create a link to the group of locations by name or by ID number. Links to location groups use "locationGroup:" in the target portion.

[link text|locationGroup:group] or just [locationGroup:group] Examples: [City Parks|locationGroup:Parks] [City Parks|locationGroup:1] [locationGroup:Parks] [locationGroup:1]

Calendars

You can link to whole calendars by name or ID number. Links to calendars use "calendar:" in the target portion.

[link text|calendar:calendar] or just [calendar:calendar] Examples: [The City Government Calendar|calendar:Government] [The City Government Calendar|calendar:3] [calendar:Government] [calendar:3]

Events

You can link to events by name or ID number. Since events can be recurring events, you can link to a specific instance of an event by including the date as well. Links to events use "event:" in the target portion.

[link text|event:event] [link text|event:date/event] or just [event:event] [event:date/event] Examples: [MPO Technical Advisory Committee Meeting|event:MPO meeting] [MPO Technical Advisory Committee Meeting|event:/2008/1/25/MPO meeting] [MPO Technical Advisory Committee Meeting|event:114] [MPO Technical Advisory Committee Meeting|event:/2008/1/25/114]

Facets

You can link to a facet by name or ID number. Links to facets use "facet:" in the target portion.

[link text|facet:facet] or just [facet:facet] Examples: [The Residents Guide|facet:Residents] [The Residents Guide|facet:3] [facet:Residents] [facet:3]
