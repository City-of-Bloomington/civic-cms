Post-Installation Setup

Once you've gone through the installation instructions, your new website is functional. Right away, though, you will want to change the administrator's user account information from the default values.

Change the Admin Account

After install, you can log in as user=administrator, pass=welcome. You will immediately want to change these values once you've logged in for the first time. The system will not work without at least one user in the system; so, you cannot delete the administrator account, but you can change all of it's information to whatever you want to log in as.

CRON scripts

The content manager ships with several bash scripts suitable for cron or command line usage. Before using them, however, they all need to be modified to provide appropriate path information for your install. Once these scripts are edited and set to be executable, they can be used in CRON. Typically, this means adding the file to /etc/cron.daily.

Backups

The system provides support for automated backups. Each backup will be a snapshot tarball of the database and data directory.

Edit /scripts/backup

  • set the path to the application
  • set the path to mysqldump
  • provide the mysql authentication information
  • set the number of days worth of backups to keep

Once it's executable you can add it to your systems' /etc/cron.daily. Because the script needs mysql username and password information, be sure to set the permissions on the script to readable only by root.
